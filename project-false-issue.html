False Issue Detection & Courier Fraud Analytics Platform

Designed and built a complete data intelligence system to detect false delivery failures (虚假问题件) by couriers in a last-mile logistics network — replacing a manual review process with algorithmic signal detection and per-agent performance dashboards. The pipeline ingests 2.3 GB of heterogeneous multi-source data, engineers 5 behavioral fraud signals using O(n log n) algorithms, and produces ranked courier leaderboards with quantified suspicion scores.

System Architecture

End-to-end pipeline from raw heterogeneous logistics data to per-agent fraud leaderboards
⬡ ETL — Ingest & Normalize
INGEST

Multi-Source ETL into 8-Table SQLite Schema

Ingests 7 UTF-8 CSVs plus 1 KMZ geospatial file (2.3 GB total) into a normalized 8-table SQLite schema with WAL journaling and a 64 MB page cache. Composite index lookups across 1.18M trajectory events reduce query latency and enable fast analytical joins between package, trajectory, and redelivery tables.

SQLite WAL64 MB Page CacheComposite Indexing7 CSVs + 1 KMZ
DEDUP

Cross-File Deduplication — 38% Record Reduction

Eliminated 38% of duplicate records (1.93M → 1.18M trajectory rows) by designing a cross-file deduplication strategy using composite (tn, op_code, op_time) keying across two overlapping source files. Ensures analytical integrity without data loss, producing a clean event sequence for downstream feature computation.

Composite Key Dedup1.93M → 1.18M rows38% Reduction
⬡ Feature Engineering — 5 Behavioral Fraud Signals
FLAGS

O(n log n) Sliding Window & Temporal Join Algorithms

All 5 feature flags are computed using NumPy searchsorted sliding windows and pd.merge_asof temporal joins — avoiding O(n²) brute-force approaches and scaling to 114,077 computable code-206 events with zero manual labeling. Each flag independently quantifies a distinct fraud behavior pattern.

NumPy searchsortedpd.merge_asofO(n log n)114,077 events
RISK

Composite Risk Scoring & Per-Agent Leaderboards

Each courier event receives a risk score equal to the number of flags triggered (0–5). Events with score ≥ 2 are classified high-risk (22,119 events). Outputs ranked per-dispatcher and per-courier leaderboards, enabling audit prioritization by quantified suspicion score rather than manual sampling.

⬡ Geospatial & Redelivery Analytics
GEO

Vectorized Haversine Proximity Engine

Parses station coordinates from KMZ/KML, then implements a vectorized Haversine distance engine (R = 6,371 km) to flag delivery events within 0.5 km of a courier's own station — indicating possible non-attempt fraud at the facility. Produces a GeoJSON-compatible proximity report for spatial audit review.

KMZ/KML ParsingHaversine (R=6371 km)0.5 km ThresholdVectorized Proximity
REDLY

Per-Agent Redelivery Analytics

Computes redelivery second-failure rates across 26,246 customer service tickets. High-volume agents (1,000+ tickets) are screened for statistically significant elevated rates using a baseline comparison, providing data-backed evidence for performance review without relying on manual sampling.

API

Package Lifecycle API — From Hours to Seconds

A queryable 02_package_lifecycle.py module reconstructs full chronological timelines per tracking number from 5 data sources, with human-readable flag explanations and JSON export. Reduces per-case investigation time from hours to seconds, enabling real-time audit support for operations teams.

5 Behavioral Fraud-Detection Feature Flags

Each flag independently signals a distinct courier fraud pattern — combined into a composite risk score
Flag 01
High-Frequency Report
Detects couriers filing an abnormally high number of false-issue reports within a short time window — a volume-based signal for batch-filing behavior.
Sliding window · NumPy searchsorted
Flag 02
Quick-Fail
Flags delivery failures filed suspiciously fast after a scan event — indicating the courier marked a package as failed without a genuine delivery attempt.
Temporal join · pd.merge_asof
Flag 03
Business-Closed Holiday
Identifies failures coded as "business closed" filed on public holidays or weekends — a calendar-based signal that the excuse is logistically implausible.
Calendar lookup · op_code match
Flag 04
Near-Station
Flags deliveries reported as failed while the GPS coordinates are within 0.5 km of the courier's own station — a geospatial signal for possible non-attempt at the facility.
Haversine · vectorized proximity · 0.5 km threshold
Flag 05 NOVEL
Burst Failure
Detects couriers who filed failures with zero successful deliveries in the surrounding ±30-minute window — surfacing a previously unmeasured batch-filing fraud pattern where entire route segments are marked failed without any genuine delivery attempt. Achieved the highest single-flag detection coverage among all five signals.
±30-minute window · highest single-flag coverage

Key Achievements

Quantified technical outcomes across the full pipeline
  • Normalized 2.3 GB of heterogeneous logistics data (7 CSVs + 1 KMZ) into a production-grade 8-table SQLite schema with WAL journaling and composite indexes — enabling sub-second analytical queries across 1.18M trajectory events.
  • Eliminated 38% of duplicate records (1.93M → 1.18M trajectory rows) via cross-file composite key deduplication, ensuring clean event sequences for downstream feature computation without data loss.
  • Engineered 5 behavioral fraud flags using O(n log n) NumPy searchsorted sliding windows and pd.merge_asof temporal joins — covering 114,077 computable events with zero manual labeling.
  • Invented the "Burst Failure" signal — detecting events where couriers filed failures with zero successful deliveries within ±30 minutes — achieving the highest single-flag detection coverage with significant unique incremental reach not captured by any existing flag.
  • Identified 22,119 high-risk courier events (risk score ≥ 2) and produced ranked per-dispatcher and per-courier leaderboards — replacing manual sampling with data-backed audit prioritization.
  • Built a geospatial proximity detector parsing station coordinates from KMZ/KML and running vectorized Haversine calculations to flag delivery events within 0.5 km of a courier's own station — a spatial signal for non-attempt fraud at the facility.
  • Delivered a queryable package lifecycle API reconstructing full timelines per tracking number from 5 sources — reducing per-case investigation time from hours to seconds.
  • Exported structured outputs — 7-sheet XLSX, Markdown reports, CSVs — including overlap matrices, risk score distributions, and dispatch-party breakdowns for non-technical stakeholder consumption.
