Agent Management & Workforce Analytics Platform

A full-stack operational analytics platform automating weekly KPI reporting for outsourced customer service agent management. Integrates a 6-step ETL orchestration pipeline ingesting 7+ heterogeneous file types, an ML call-volume predictor (R²>0.90), a 9-rule 4-week rolling trend classification engine, a Mixed-Integer Linear Program (MILP) for shift scheduling, and a 5-layer fuzzy name resolution system — all served through a Flask web interface with interactive Plotly and Chart.js dashboards.

System Architecture

6-step ETL feeding 6 analytical modules, served through a Flask web interface
⬡ Data Layer — ETL & Name Resolution
ETL

6-Step ETL Orchestration — 7+ File Types

Ingests 7+ heterogeneous weekly file types (QP CSVs, Udesk/WhatsApp Excel exports, attendance reports) through a 6-step ETL pipeline and persists results into a normalized SQLite schema across 10+ SQLAlchemy models. Each step is independently rerunnable, with schema migration handled via SQLAlchemy ORM.

7+ File Types10+ ORM ModelsSQLAlchemySQLiteWeekly Orchestration
FUZZ

5-Layer Fuzzy Name Resolution — <5% Unmatched

Resolves agent names across Hotline, Udesk, and WhatsApp QA channels using a cascade of five matching strategies: exact match → word-subset → name-swap → FuzzyWuzzy token ratio (threshold = 0.7) → manual fallback. Handles real-world variations like "Raquel Alberto Cazas" ↔ "Raquel Cazas", achieving <5% unmatched rate across all channels without manual correction.

FuzzyWuzzyThreshold = 0.75-Layer Cascade<5% Unmatched
⬡ Intelligence Layer — ML, Rules & Optimization
ML

Call-Volume Predictor — scikit-learn Linear Regression, R²>0.90

Trains a scikit-learn linear regression model on package volume and AI control rate features to predict weekly hotline call volume. Achieves R²>0.90 on held-out test data. Deployed as a Flask API endpoint with interactive 3D Plotly surface visualizations for stakeholder reporting — mapping the call-volume response surface across feature combinations.

scikit-learnLinear RegressionR²>0.90Flask API3D Plotly Surface
RULES

9-Rule, 4-Week Rolling Trend Classification Engine

Auto-classifies agents into Good / Warning / Disqualified tiers using a 9-rule engine evaluated over a 4-week rolling window. Rules fire on core KPI failure thresholds, service incident counts, and attendance patterns — replacing manual spreadsheet review with deterministic, auditable logic that produces a weekly ranked tier report.

9 Classification Rules4-Week Rolling Window3-Tier OutputKPI Thresholds
MILP

Mixed-Integer Linear Program Shift Scheduler (PuLP)

Formulates and solves a MILP using PuLP to generate minimum-headcount shift schedules covering a 12-hour operational window (09:00–21:00). Enforces split-shift constraints (≥1-hour break, 8-hour total shift), and outputs actionable Excel rosters. Objective: minimize total agents scheduled while satisfying per-hour demand constraints.

PuLP MILPInteger Programming09:00–21:00 WindowSplit-Shift ConstraintsExcel Roster Output
⬡ Presentation Layer — Dashboards & Reporting
DASH

Work Order Analytics — Chart.js Drill-Down Dashboards

Surfaces multi-dimensional work order trends across multiple source categories and geographic regions through interactive Chart.js drill-down dashboards with server-side aggregation in Flask. Enables data-driven weekly review cycles for non-technical operations managers without SQL access.

Generates per-agent performance dashboards covering Hotline, Udesk, and WhatsApp QA channels, with weekly XLSX exports including agent tier rankings, KPI breakdowns, and attendance summaries — enabling managers to run performance reviews entirely from the platform without manual data collection.
