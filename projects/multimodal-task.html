Multimodal Task Management System

👥 Group 5 · Xindan Zhang · Yixin Zhang · Yunning Chen · Ziyu Chen

A full-stack student productivity platform that uses Vision-Language Models (VLMs) and LangChain to automatically "read" unstructured assignment screenshots and parse them into structured task metadata — no manual entry required. Architected with a React frontend, Flask RESTful API, and SQLite backend, and designed end-to-end from a 15-screen Figma prototype validated through formal user research with 8 participants.

System Architecture

Three-layer full-stack pipeline — Vision AI extraction → Business logic → React UI

// Layer 1 · Multimodal Input — VLM Screenshot Extraction
INPUT

Screenshot Upload

User uploads an unstructured image — a Brightspace assignment page, course syllabus screenshot, or any academic deadline document — through a drag-and-drop React UI component.

React Multimodal input Unstructured data
VLM

Vision-Language Model Extraction

The image is passed to OpenAI Vision API via a LangChain chain. The VLM "reads" the visual content and extracts structured fields: task name, course, due date, submission URL, and any additional notes — with no template or fixed format required on the input side.

OpenAI Vision API LangChain Structured extraction Zero-shot parsing
PARSE

Metadata Structuring

LangChain output parsers normalize the VLM response into a typed task schema — validating date formats, cleaning URLs, and flagging low-confidence extractions for manual review before they reach the database.

Output parsing Date normalization Confidence flagging
// Layer 2 · Backend — Flask RESTful API + SQLite
API

Flask RESTful Endpoints

RESTful API layer handles task CRUD operations, screenshot upload routing, AI chatbot queries, and notification management. Designed to decouple the React frontend from AI processing logic.

Flask REST API CRUD File upload
DB

SQLite Task Store

Lightweight persistent storage for tasks, deadlines, progress states, submission links, and user preferences. Schema supports task grouping, custom ranking, and per-task notes.

SQLite Task schema Progress tracking
// Layer 3 · Frontend — React UI
UI

React Frontend — One-Click Task Creation

The "secret recipe" of the product: a single upload interaction creates a fully populated task entry. The UI renders both a sortable list view and a calendar view, lets users manually edit any field, and links directly to submission pages.

React List view Calendar view One-click entry
AI

Embedded AI Chatbot

An in-app LLM chatbot answers questions about pending deadlines, suggests prioritization based on workload, and provides study time recommendations — rebased from the original "AI time estimator" concept after user research revealed skepticism about automated accuracy.

LLM chatbot AI prioritization Weekly summary

Product Features

Eight capabilities defined through customer discovery interviews and iterative design

📸
Feature 01 · Secret Recipe
One-Click Screenshot Task Import
Upload any assignment screenshot — Brightspace, Gradescope, course website, syllabus — and the VLM pipeline auto-populates every task field. Validated as the highest-priority feature across all user research sessions.
VLMs · LangChain · zero manual entry
🤖
Feature 02
AI Chatbot & Smart Recommendations
Conversational LLM interface for deadline queries, workload prioritization, and study time suggestions. Evolved from a deterministic "AI time estimator" to a transparent chatbot format after user research revealed skepticism of black-box AI predictions.
LLM · interactive · transparent
📅
Feature 03
Dual-View Task Management
Sortable, groupable list view with user-defined ranking criteria alongside a calendar view mapping due dates visually. Supports manual editing of any AI-extracted field for correction and customization.
list view · calendar view · custom sort
🔔
Feature 04
Customizable Notifications & Reminders
Configurable deadline reminders with per-task notification schedules. A notification archive preserves all past alerts for reference. Addresses the core discovery finding that students miss deadlines due to fragmented platform checking.
per-task scheduling · archive
🔗
Feature 05
Direct Submission Links
Each task stores and displays a direct link to the assignment submission page — extracted automatically from the screenshot when available. Identified as a high-interest feature that eliminates the multi-platform navigation pain point.
auto-extracted · one-click submit
📊
Feature 06
Progress Tracking & Weekly Summary
Percentage-based progress notation per task with a weekly AI-generated learning pattern summary. Completion milestones award coins redeemable for UI themes (dark, forest, ocean) — gamifying productivity.
progress % · gamified rewards · AI summary

User Research & Hypothesis Testing

8 participants · 5 hypotheses tested · findings drove major design pivots

H1 · Value Proposition ✓ SUCCESS
Centralized Task Management
Strong interest in a unified platform. Discovery interviews confirmed students juggle 4–6 platforms (Brightspace, Gradescope, DEN, GitHub, professor sites) — validating the core need. Exceeded the 70% interest threshold.
H2 · Business Model ↻ REVISED
Subscription Pricing
Users preferred free + ads over paid subscription. Business model pivoted to freemium — core features free, premium AI features (deeper analysis, weekly reports, knowledge gap identification) behind a paid tier.
H3 · One-Click Entry ✓ HIGH PRIORITY
Screenshot-to-Task Import
Overwhelming support across all participants. Designated the product's "secret recipe" — the single differentiating capability that no direct competitor (Todoist, Google Calendar, Notion, Motion) offers for academic deadlines.
H4 · AI Estimation ↻ REBRANDED
AI Time Estimator → AI Chatbot
Mixed feedback — users skeptical of AI accurately estimating task duration without context. Reframed as an interactive AI chatbot emphasizing conversation and transparency over automated prediction, improving perceived trustworthiness.
H5 · Submission Link ✓ SUCCESS
In-App Submission Link
High interest from all participants. Direct submission links eliminate the need to re-navigate to the correct platform page at submission time — a common friction point. Promoted to a prominently showcased feature in the demo.

Design & Engineering Highlights

Key decisions made through customer discovery, competitive analysis, and iterative prototyping

  • VLM zero-shot extraction — no fixed template or academic platform integration required; the model generalizes across any screenshot layout, making the pipeline platform-agnostic from day one.
  • Discovery-driven design — three customer interviews (Bioengineering, Biomedical Eng, Applied Math students) surfaced the exact platforms causing friction; all five features were directly traced to interview findings.
  • Competitive gap analysis — benchmarked against 12 competitors (Todoist, Notion, Motion, Sunsama, ClickUp, Trello, D2L Brightspace Pulse, etc.); the academic-specific AI import capability was an unaddressed gap across all.
  • Gamified completion loop — coins earned from task submissions unlock purchasable UI themes (dark, forest, ocean), creating a positive reinforcement cycle for on-time submission behavior.
  • AI chatbot pivot — replaced a prescriptive "AI time estimator" with a conversational interface after discovering user skepticism of black-box AI outputs; the interactive format restored user trust and perceived accuracy.
  • Freemium business model — post-research pivot to free + ads for core features, with premium AI deep analysis (weekly learning patterns, knowledge gap identification) as the monetization layer.
