Core Features

Six capabilities that define the system's trading and research workflow

📡
Feature 01
W-Bottom Pattern Scanner
Scans a configurable stock universe (up to S&P 500) for W-shaped double-bottom reversals. Detects two local minima within a 60-bar lookback window, validates bottom symmetry within 5% tolerance, checks 10–40 bar separation, and confirms breakout above the neckline with an optional 1.2× volume spike filter. Emits a BUY signal with breakout percentage and pattern metadata.
lookback 60 bars · vol confirm 1.2×
🧠
Feature 02
OpenClaw AI Stock Analysis
Three parallel research skills — Technical (SMA 20/50/200, RSI 14, Bollinger Bands, price patterns), Fundamental (P/E, margins, ROE/ROA, debt/equity, growth), and News Sentiment (headline classification) — each build a prompt from live market data and invoke Claude for a structured analysis with a confidence score. The orchestrator's /manifest endpoint allows OpenClaw to discover and chain skills autonomously.
3 skills · parallel async · confidence scored
⚙️
Feature 03
Automated Sell Rule Engine
Six builtin monitor rules evaluated against every open position on each tick. Rules bind to specific portfolios or symbols and fire sell signals automatically when triggered. Custom rules can be written in Python, validated via AST, and hot-reloaded — no restart needed. Composite AND/OR/NOT logic supported for complex multi-condition exits.
6 builtin rules · sandbox · hot-reload
🔄
Feature 04
Dual Interface — Human & AI
Every system function is accessible through three surfaces: a Click CLI with JSON output for scripting and AI parsing, a 36-route FastAPI backend for programmatic control, and a Next.js web UI for human interaction. The /api/v1/manifest endpoint publishes a machine-readable capability map so OpenClaw can introspect available commands and call them with minimal guidance.
CLI · REST API · Web UI · AI manifest
📊
Feature 05
Strategy SDK + Backtesting
Four builtin strategies (W-Bottom, Momentum RSI, Mean Reversion Z-score, Trend-Following Donchian, Dual Momentum) plug into a common BaseStrategy ABC. The event-driven backtest engine replays bars through SimulatedBroker with the same risk checks as live mode. Performance analyzer surfaces Sharpe ratio, Sortino ratio, max drawdown, and win rate in a Rich-formatted CLI report.
4 strategies · same risk path as live
🛡️
Feature 06
Risk Management & Audit Trail
Pre-trade validation enforces position size limits, portfolio concentration caps, and daily loss thresholds before any order reaches the broker. Every signal, order submission, fill, rejection, and rule trigger is written to an append-only audit log, enabling post-trade attribution and compliance review.
pre-trade checks · append-only audit log

Automated Position Monitoring Rules

Six builtin rules that fire sell signals or alerts automatically on each position tick

trailing_stop
Sells when price drops below a percentage from the highest observed price since entry. Tracks a rolling high-watermark.
Default: 5% trail · severity: critical
take_profit
Sells when unrealized gain from average cost reaches the configured target percentage.
Default: 20% gain target · severity: warning
max_holding_days
Forces an exit after a position has been held for more than N calendar days, preventing indefinite tie-up.
Default: 90 days · severity: warning
volume_anomaly
Alerts when intraday volume exceeds a configurable multiple of the 20-day average — signals unusual institutional activity.
Default: 2.5× 20-day avg · alert only
rsi_divergence
Alerts when price and RSI-14 are diverging — price making new highs while RSI declines, or vice versa.
RSI period: 14 · alert only
earnings_proximity
Alerts when a scheduled earnings date is within the warning window, prompting a position review before event risk.
Default: 7-day warning · alert only
  • Custom rules can be written in Python using the BaseMonitorRule ABC and deployed live via the Monaco editor in the web UI — no server restart required.
  • Composite logic supports AND/OR/NOT combinations of any rules for multi-condition exit strategies.
  • Rule scope can be bound to an entire portfolio or limited to specific symbols, giving fine-grained control.
  • Alert dispatch routes to terminal output, WebSocket push, Slack webhook, or email — configurable per severity.

Design Highlights

Engineering decisions that make the system extensible and AI-ready

  • AI-native manifest endpoint — GET /api/v1/manifest returns a machine-readable capability map, allowing OpenClaw to discover available strategies, skills, and commands without prior knowledge of the API.
  • Unified broker interface — swapping from simulated to paper to live Alpaca is a one-line config change; all strategy and monitoring code runs identically across modes.
  • Sandboxed rule execution — custom Python rules undergo AST-level validation before exec(), blocking dangerous imports and builtins; enables safe user-defined logic in a web editor.
  • Full async stack — FastAPI, SQLAlchemy 2.0 (async), and data providers all use asyncio; the engine's asyncio.TaskGroup runs strategy loops, monitoring, and snapshot refreshes concurrently.
  • Three-surface access — every operation available on CLI, REST API, and web UI ensures OpenClaw can call any function while human traders have a visual interface for the same workflows.
  • Plugin architecture — strategies, monitor rules, and research skills auto-discovered at startup via pkgutil; adding a new strategy requires only a new file in the builtin/ directory.
