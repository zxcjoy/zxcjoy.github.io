VOC Intelligence & Competitive Analysis Platform

An end-to-end Voice-of-Customer intelligence system that transforms unstructured customer reviews into actionable competitive intelligence. The platform scrapes 100K+ posts across 14 platforms, classifies them with LLMs across a 110-category issue taxonomy, and generates Claude Sonnet-powered per-company competitive analysis reports — complete with AI root-cause narratives, prioritized recommendations, and review authenticity assessments.

System Architecture

Fully decoupled, phase-based pipeline — Scrape → Analyze → Report
⬡ Phase 1 — Scrape
SCRAPER

Multi-Platform Web Scraping (14 Sources)

Collects posts and comments from Reddit, Trustpilot, BBB, YouTube, TikTok, and 7 Chinese-language platforms via Playwright-driven browser automation. Checkpoint-based resumability ensures zero data loss on interruption or machine restart — pipelines can be stopped mid-run and resumed from exactly where they left off.

Playwright14 PlatformsCheckpoint Resumability35+ Companies
DEDUP

SimHash Near-Duplicate Detection & Language ID

Before LLM inference, applies SimHash-based near-duplicate detection to filter semantically identical posts across platforms, and language identification (EN/ZH) to route content appropriately. Reduces noise and LLM token spend on irrelevant or repeated content.

SimHashLanguage ID (EN/ZH)Preprocessing
⬡ Phase 2 — Analyze
LLM

3-Stage LLM Classification Pipeline

Uses MiniMax-M2.5 (accessed via Anthropic SDK) in a 3-stage pipeline: (1) sentiment scoring per post, (2) primary issue category detection, (3) multi-label leaf-category classification across 110 categories in the issue taxonomy. Batches 5 posts per prompt call — achieving a 5× API cost reduction critical for subscription-based token pricing.

MiniMax-M2.5Anthropic SDK3-Stage Pipeline5-Post Batching110-Category Taxonomy
STORAGE

Concurrent SQLite with WAL Mode

All classified posts are stored in SQLite with WAL (Write-Ahead Logging) mode enabled for concurrent write access across 8 threads. Parameterized, case-insensitive company resolution supports any of 35+ tracked companies via a single CLI flag.

SQLite WAL8 ThreadsConcurrent WritesClick CLI
⬡ Phase 3 — Report
REPORT

Claude Sonnet-Powered Competitive Analysis

Generates per-company HTML and XLSX competitive analysis reports via Claude Sonnet. Each report includes: AI root-cause narratives for top issues, prioritized P0/P1/P2 recommendations with effort and timeline estimates, review authenticity assessments for top-rated competitors, and a company vs. industry sentiment comparison. Response caching eliminates redundant API calls on re-runs.

Claude SonnetHTML / XLSX ExportRoot-Cause AnalysisAuthenticity AssessmentResponse Caching

Key Achievements

Quantified technical outcomes
  • 100K+ posts collected across 14 platforms (Reddit, Trustpilot, BBB, YouTube, TikTok + 7 Chinese platforms), covering 35+ US logistics companies with zero data loss via checkpoint-based resumability.
  • 5× API cost reduction through 5-post-per-prompt batching in the LLM classification pipeline — critical for subscription-based token pricing at scale.
  • SimHash dedup + language ID as preprocessing steps reduced noise in downstream LLM inference by filtering near-duplicate and irrelevant content before classification.
  • 110-category issue taxonomy enabling multi-label classification of delivery complaints, tracking failures, customer service breakdowns, and package security issues across the full industry.
  • Claude Sonnet reporting layer generates per-company vs. industry HTML/XLSX reports with AI root-cause narratives and review authenticity assessments, with response caching to eliminate redundant API calls.
  • Fully decoupled 3-phase architecture (Scrape → Analyze → Report) with 8-thread concurrent execution and SQLite WAL-mode write concurrency — enabling any of 35+ companies to be analyzed via a single CLI flag.

Root Cause Analysis

Inadequate driver training and verification protocols for address confirmation at point of delivery.

FedEx drivers appear to be under time pressure to complete high-volume routes, leading to insufficient address verification before delivery. The company likely lacks robust technological tools or protocols that force drivers to confirm correct addresses through photo evidence or GPS verification before marking packages as delivered.

  • High delivery volume pressure leading to rushed deliveries
  • Insufficient use of address verification technology
  • Lack of standardized photo documentation requirements

Recommendations

P0 Implement mandatory photo documentation showing package placement AND visible address numbers for all residential deliveries Immediate P0 Deploy GPS-based address verification requiring driver confirmation when delivery location doesn't match package address coordinates 1–3 months P1 Revise driver performance metrics to prioritize delivery accuracy over speed, with penalties for confirmed wrong-address deliveries Immediate

Competitor Insight — UPS: UPS maintains significantly better address accuracy at 1.1% vs FedEx's 8.1%, likely through better driver training and delivery verification systems. FedEx performs 7× worse than UPS — indicating systemic operational deficiencies rather than an industry-wide challenge.
